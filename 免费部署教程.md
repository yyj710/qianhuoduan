# 免费部署教程 — 0元让全世界访问你的网站

> 全部免费，不需要信用卡，不需要花钱。

---

## 整体架构

用 4 个免费服务拼起来：

```
用户浏览器
    ↓
  Vercel（前端页面，免费）
    ↓ 调用 API
  Render（后端程序，免费）
    ↓ 读写数据
  Neon（PostgreSQL 数据库，免费）
```

| 平台 | 做什么 | 免费额度 | 
|------|--------|----------|
| **GitHub** | 存放代码 | 无限 |
| **Neon** | 数据库（存用户、技能、订单） | 0.5GB，够用 |
| **Render** | 跑后端程序 | 750小时/月 |
| **Vercel** | 展示前端页面 | 无限流量 |

## 第一步：注册所有平台账号

### 1.1 注册 GitHub（放代码）

1. 打开 👉 **https://github.com**
2. 点右上角「Sign up」
3. 填用户名、邮箱、密码
4. 用户名建议取个简单英文名，比如 `chuangyin2026`
5. 验证邮箱（去邮箱点确认链接）

### 1.2 注册 Neon（数据库）

1. 打开 👉 **https://neon.tech**
2. 点「Sign Up」
3. 选「Continue with GitHub」（用刚注册的 GitHub 账号登录）
4. 允许授权即可

### 1.3 注册 Render（后端）

1. 打开 👉 **https://render.com**
2. 点「Get Started」
3. 选「Sign in with GitHub」
4. 允许授权即可

### 1.4 注册 Vercel（前端）

1. 打开 👉 **https://vercel.com**
2. 点「Sign Up」
3. 选「Continue with GitHub」
4. 允许授权

> ✅ 四个平台都注册好了，全用同一个 GitHub 账号登录，互相打通。

## 第二步：把代码上传到 GitHub

### 2.1 下载 GitHub Desktop

👉 **https://desktop.github.com**

点「Download for Windows」，安装。

### 2.2 创建仓库

1. 打开 GitHub Desktop
2. 用你的 GitHub 账号登录
3. 点「Create a New Repository」（创建新仓库）
4. 这样填：
   - Name：`chuangyin`（或随便取名）
   - Local Path：选你桌面的 `网站前后端` 文件夹
   - 勾选「Initialize with a README」：**不勾**
5. 点「Create Repository」

### 2.3 上传

1. 左下角「Summary」随便填：`初始化项目`
2. 点「Commit to main」
3. 点「Publish repository」
4. 弹出窗口，**取消勾选**「Keep this code private」（选公开）
5. 点「Publish Repository」

> ✅ 代码已经上传到 GitHub。打开 https://github.com/你的用户名/chuangyin 就能看到。

---

## 第三步：创建 Neon 数据库

### 3.1 新建数据库

1. 打开 👉 **https://console.neon.tech**
2. 点「Create a project」
3. 这样填：
   - Project name：`chuangyin-db`
   - Region：选离深圳最近的，比如「Asia Pacific (Singapore)」或「Asia Pacific (Tokyo)」
4. 点「Create project」
5. 等约 30 秒，创建完成

### 3.2 获取数据库连接地址

1. 创建完成后，你会看到一个「Connection Details」区域
2. 找到「**Connection string**」一栏
3. 它长这样：
   ```
   postgresql://chuangyin_xxx:AbCdEf123456@ep-cool-sky-123456.ap-southeast-1.aws.neon.tech/chuangyin-db?sslmode=require
   ```
4. 点旁边的 **复制按钮**，**保存到记事本**，后面要用！

> 📝 **重要！** 这个连接字符串就是数据库的钥匙，**好好保存**。

## 第四步：部署后端到 Render

### 4.1 创建后端服务

1. 打开 👉 **https://dashboard.render.com**
2. 点顶部「New +」→「Web Service」
3. 点「Connect」连接你的 `chuangyin` 仓库

### 4.2 配置服务

填下面的表单：

| 字段 | 填什么 |
|------|--------|
| **Name** | `chuangyin-api` |
| **Region** | `Singapore`（离深圳近） |
| **Root Directory** | `backend` |
| **Runtime** | `Node` |
| **Build Command** | `npm install && npx prisma generate && npx prisma db push && npm run build` |
| **Start Command** | `npm start` |
| **Instance Type** | **Free** |

### 4.3 添加环境变量

往下翻到「Environment Variables」区域，添加 4 个：

| Key | Value |
|-----|-------|
| `DATABASE_URL` | 粘贴第三步复制的 Neon 连接字符串 |
| `JWT_SECRET` | 你自己瞎编的一串乱码，越长越好，比如 `wodeBGchuangyin2026!@#Abc` |
| `FRONTEND_URL` | 暂时填 `*`（后面拿到 Vercel 地址再改） |
| `PORT` | `3000` |

### 4.4 启动

1. 点最下方「Deploy Web Service」
2. 等约 3-5 分钟（第一次比较慢，要下载依赖、编译代码）
3. 看到「Live」绿色标签，说明跑起来了

### 4.5 记下后端地址

部署完成后，页面顶部会显示一个地址：
```
https://chuangyin-api.onrender.com
```
**复制下来，存到记事本！**

---

## 第五步：部署前端到 Vercel

### 5.1 导入项目

1. 打开 👉 **https://vercel.com/new**
2. 找到你的 `chuangyin` 仓库，点「Import」

### 5.2 配置

1. **Root Directory**：点「Edit」，选择 `frontend`
2. **Framework Preset**：默认就是 Vite，不用改
3. **Build Command**：默认 `npm run build`，不改
4. **Output Directory**：默认 `dist`，不改

### 5.3 添加环境变量

点「Environment Variables」，添加：

| Key | Value |
|-----|-------|
| `VITE_API_URL` | 粘贴第四步记下的后端地址 + `/api`，像这样：`https://chuangyin-api.onrender.com/api` |

### 5.4 部署

点「Deploy」，等约 1 分钟。

部署完成后，Vercel 会给你一个地址：
```
https://chuangyin.vercel.app
```

### 5.5 回填前端地址到 Render

1. 回到 Render 控制台
2. 点 `chuangyin-api` 服务
3. 左侧点「Environment」
4. 修改 `FRONTEND_URL` 为 `https://chuangyin.vercel.app`
5. 点「Save Changes」，会自动重启服务

## 常见问题

### Q：部署失败了怎么办？
打开对应的平台控制台，找到「Logs」或「Deploy Logs」，看红色报错信息。大多数情况是环境变量没填对。

### Q：网站打开是空白？
按 F12，看 Console 里的报错。通常是 `VITE_API_URL` 环境变量没填或填错了。

### Q：免费额度用完怎么办？
- Render 每月 750 小时，刚好够一个月跑满（一个月只有 730 小时）
- Neon 免费 0.5GB 存储，几百个用户足够
- Vercel 免费额度远超需求
- **正常使用不会超额**

### Q：免费服务会关掉我的网站吗？
- Render 免费版：如果 15 分钟没人访问，服务会"休眠"。下一个用户访问时会等 30-50 秒唤醒。
- 解决办法：后面用户多了，再考虑付费（7美元/月，不休眠）

### Q：想用自己的域名怎么办？
Vercel 设置里可以绑定自定义域名。需要先买域名（约30元/年），然后改 DNS 解析到 Vercel。具体操作以后可以问我。

### Q：网站能支持多少人？
免费方案适合 **几百到几千用户**。足够深圳大学内部使用了。
