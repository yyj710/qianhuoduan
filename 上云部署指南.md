# 深技联 — 让网站上线，别人也能访问

> 用最通俗的话解释每一步是干什么的，为什么要做。

---

## 先搞清楚：为什么现在只有你能访问？

你的网站现在跑在**你自己的电脑上**（localhost），就像你在自己房间里开了个小店——

- `localhost:5173` = 你家门牌号，只有你自己能进
- 别人在别的地方，找不到你家

要让别人也能来，你需要：
1. **租个"商铺"**（云服务器）—— 一个24小时不关机的公共电脑
2. **把网站搬上去**（部署）—— 把代码和数据放上去
3. **挂个"招牌"**（域名）—— 让别人好找好记

---

## 第一步：租个云服务器（买"商铺"）

### 这是什么？
云服务器就是一台**永远不关机的电脑**，放在腾讯/阿里的机房里，有公网IP地址（相当于公开的街道地址），全世界都能访问。

### 怎么做？
去 **腾讯云** 或 **阿里云** 官网，搜"轻量应用服务器"或"学生服务器"：
- 配置：2核CPU + 4G内存（够几百人用了）
- 系统选：**Ubuntu 22.04**（一种Linux操作系统，免费）
- 价格：学生优惠大约 **50元/月**，一年600块

买完你会得到：
- 一个 **IP地址**（像 `119.29.xxx.xxx`，服务器的网络地址）
- 一个 **root密码**（管理员密码，登录服务器用）

> 💡 **比喻**：你租了个商铺，拿到了门牌号和钥匙。

---

## 第二步：安装运行环境（装修店铺）

云服务器刚买来是个空壳子，什么都没装。你需要装上能让网站跑起来的软件。

用工具（如 Xshell、Termius 或腾讯云网页终端）连接到服务器后，依次敲下面这些命令：

### 2.1 安装 Node.js — 网站的"发动机"
```
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```
> **解释**：你的后端代码是用 Node.js 写的，就像汽油车需要汽油发动机。装了这个，服务器才能看懂并运行你的代码。

### 2.2 安装 Nginx — 网站的"前台接待"
```
sudo apt-get install -y nginx
```
> **解释**：Nginx 是世界上最流行的网页服务器软件。它做两件事：
> - 把网站页面"端"给来访的用户
> - 当用户访问 `/api/xxx` 时，把请求转给后端处理（叫"反向代理"）
>
> 就像餐厅前台——顾客来了先找前台，前台把点菜需求传给后厨。

### 2.3 安装 PM2 — 网站的"保安"
```
sudo npm install -g pm2
```
> **解释**：你的后端是一个 Node.js 程序。如果直接运行，一旦窗口关了或出错了，程序就停了。PM2 是进程守护工具，作用：
> - 保证程序24小时运行
> - 程序崩溃了自动重启
> - 服务器重启后自动启动你的程序
>
> 就像保安24小时守着店，停电了来电后自动开门。

### 2.4 安装 Git — 代码"搬运工"
```
sudo apt-get install -y git
```
> **解释**：用来把你电脑上的代码传到服务器上。也可以直接用 FTP 工具传。

## 第三步：把代码传上去（把货搬进店铺）

### 3.1 在服务器上创建目录
```
mkdir -p /www/chuangyin
cd /www/chuangyin
```
> **解释**：在服务器上新建一个文件夹，专门放你的项目。

### 3.2 上传代码

**最简单的方法**：用 FTP 工具（推荐 FileZilla，免费）

把你桌面 `创英第二主题` 文件夹里的 `backend` 和 `frontend` 两个文件夹，拖到服务器的 `/www/chuangyin/` 目录下。

> **解释**：就是把文件从你电脑拷到服务器上，跟 U 盘拷文件一样。

---

## 第四步：启动后端（让后厨开始干活）

进入后端文件夹，一句一句敲：

### 4.1 安装依赖包
```
cd /www/chuangyin/backend
npm install
```
> **解释**：你的代码引用了很多第三方库（Express、Prisma等），这些库的代码不在你的项目文件夹里（太大了）。这个命令根据 `package.json` 里列出的清单，自动下载所有需要的库。

### 4.2 设置密码（非常重要！）
```
cat > .env << 'EOF'
PORT=3000
DATABASE_URL="file:./dev.db"
JWT_SECRET="换成你自己瞎编的一串乱码，越长越好比如 chuanyin2026!@#$%ABcdEFgh"
EOF
```
> **解释**：`.env` 是环境变量配置文件。
> - `PORT`：后端监听的端口号
> - `DATABASE_URL`：数据库文件位置
> - `JWT_SECRET`：**安全密钥**。用户的登录状态用这个密钥加密签名。现在是 `chuangyin_jwt_secret_2026`，太好猜了！**必须改成自己瞎编的长串**，否则别人能伪造登录信息。

### 4.3 初始化数据库
```
npx prisma migrate deploy
npx prisma generate
```
> **解释**：Prisma 是帮你操作数据库的工具。这两条命令：
> - 第一条：按照你的表结构设计，在服务器上创建数据库文件（SQLite，一个 .db 文件）
> - 第二条：生成数据库操作的代码，让后端能读写数据库

### 4.4 编译并启动
```
npm run build
pm2 start dist/index.js --name chuangyin-api
pm2 save
pm2 startup
```
> **解释**：
> - `npm run build`：把 TypeScript 代码**翻译**成 Node.js 能直接运行的 JavaScript
> - `pm2 start`：用 PM2 启动你的后端程序，起个名字叫 chuangyin-api
> - `pm2 save`：保存当前运行状态
> - `pm2 startup`：设置开机自启，服务器重启后自动恢复

---

## 第五步：发布前端（把店面装修好展示出去）

### 5.1 编译前端
```
cd /www/chuangyin/frontend
npm install
npx vite build
```
> **解释**：
> - 前端 React 代码在开发时有很多调试信息，体积很大
> - `vite build` 把代码**压缩打包**成一个优化的 `dist` 文件夹，体积小、加载快，这才是给用户看的版本

### 5.2 配置 Nginx（告诉前台怎么接待客人）
```
sudo tee /etc/nginx/sites-available/chuangyin << 'EOF'
server {
    listen 80;
    server_name 你的域名或IP;

    root /www/chuangyin/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
EOF
```
> **解释**：这是 Nginx 的"工作手册"，告诉它遇到不同的请求怎么做：
> - 用户访问首页 `/` → 给前端页面（dist 文件夹里的 index.html）
> - 用户访问 `/api/xxx` → 转给后端（3000端口）处理
> - `try_files` 这行的作用是：React 是单页面应用，URL 切换由前端自己处理，刷新页面时不报404

### 5.3 启用配置
```
sudo ln -s /etc/nginx/sites-available/chuangyin /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```
> **解释**：
> - 第一条：把刚才写的"工作手册"正式启用（创建软链接）
> - 第二条：检查配置是否正确，正确的话重新加载 Nginx

## 第六步：配 HTTPS（给店铺装防盗门）

```
sudo apt-get install -y certbot python3-certbot-nginx
sudo certbot --nginx -d 你的域名
```
> **解释**：
> - HTTP = 明文传输，数据在路上可能被偷看
> - HTTPS = 加密传输，安全。浏览器地址栏会显示🔒
> - certbot 是 Let's Encrypt（全球免费SSL证书组织）的工具，自动帮你申请和安装证书
> - 这是**全免费的**，不需要花钱买证书

---

## 第七步：买域名（挂招牌）

IP 地址是一串数字（119.29.xxx.xxx），不好记。域名就是给它起个好记的名字。

去哪儿买：**腾讯云** 或 **阿里云** 搜"域名注册"
- 选个 `.com` 或 `.cn` 结尾的，比如 `szskill.com`
- 价格：普通域名约 **30-60元/年**
- 买完后，在域名管理里把域名**解析**到你的服务器 IP

> **解释**：域名解析 = 告诉全世界的网络："szskill.com 这个门牌对应 119.29.xxx.xxx 这个地址"。

## 通俗流程回顾

```
1. 买服务器       →   租了个店铺，拿到地址和钥匙
2. 装软件         →   装修：通水电（Node.js）、设前台（Nginx）、请保安（PM2）
3. 传代码         →   把货搬进去
4. 启动后端       →   后厨开始干活
5. 发布前端       →   店面开张迎客
6. 配 HTTPS       →   装防盗门
7. 买域名         →   挂招牌
```

**做完这些，任何人在浏览器输入你的域名，就能注册、登录、使用你的网站了。**

## 几个你可能想问的问题

**Q: 我必须一次性做完所有步骤吗？**
不用。你可以先买服务器 + 做完部署，域名和 HTTPS 后面再加。

**Q: 如果网站出了问题怎么办？**
用 `pm2 status` 看看后端是不是在跑，`pm2 logs` 看日志。Nginx 用 `sudo nginx -t` 检查配置。

**Q: 用户数据存在哪？**
存在服务器的 `backend/prisma/dev.db` 文件里（SQLite 数据库）。**记得定期备份这个文件**。

**Q: SQLite 够用吗？**
几百人日常用完全够了。如果用户量上千了，再换成 MySQL 或 PostgreSQL（改一行配置就行）。
